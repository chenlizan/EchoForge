# EchoForge 产品使用说明书

EchoForge 是一个面向因果分析与研究探索的可视化工作台。你可以把数据、图表、因果图、估计结果和人工笔记都放在同一张无限画布上，通过节点之间的关系逐步推进分析。

本说明书面向产品使用者与部署者，包含启动方式、核心概念、典型操作流程和常见问题。

## 1. 产品定位

EchoForge 适合以下场景：

- 导入结构化数据后快速生成探索性分析图表
- 为变量分配角色并推断因果图
- 基于因果图生成识别与效应估计结果
- 在分析过程中插入人工笔记、假设和研究判断
- 通过节点分叉保留不同分析路径

## 2. 核心概念

### 2.1 画布

产品主界面是一张可平移、可缩放的无限画布。所有分析结果都以节点形式放在画布上，通过连线表达来源关系和分析流程。

### 2.2 节点类型

- `Dataset`：数据集节点，承载导入的数据表和变量角色
- `Visualization`：可视化节点，展示散点图、分布图等探索分析结果
- `Causal Graph`：因果图节点，展示变量之间的因果结构
- `Insight`：分析结论或人工笔记节点，也用于承载估计结果

### 2.3 连线关系

节点之间的关系由箭头连线表示，常见语义包括：

- `eda`：由数据集派生出的探索分析
- `models`：由数据集生成的因果图
- `identifies`：由因果图生成的识别与估计结果
- `analysis` / `action`：由已有结论继续扩展出的后续分析
- `explains`：由图表中的异常点分析生成的解释节点
- `fork`：从现有分析分叉出的独立分支

### 2.4 手动节点

界面中的 `Manual Node` 本质上会创建一个 `Insight` 节点，用于记录人工笔记、假设、研究判断或待办事项。它既可以单独存在，也可以作为后续 AI 分析的输入上下文。

## 3. 系统要求

运行本项目建议具备以下环境：

- Node.js
- Python 3.8+
- [Ollama](https://ollama.com/) 本地服务

推荐模型：

- `gemma3:12b`

## 4. 本地启动

### 4.1 安装前端依赖

```bash
npm install
```

### 4.2 启动 Ollama

确保本地 Ollama 服务已运行，默认地址：

```bash
http://127.0.0.1:11434
```

拉取模型：

```bash
ollama pull gemma3:12b
```

### 4.3 启动后端服务

进入后端目录并安装依赖：

```bash
cd server
pip install -r requirements.txt
```

创建 `server/.env`：

```env
OLLAMA_BASE_URL=http://127.0.0.1:11434
OLLAMA_MODEL=gemma3:12b
PORT=5000
```

启动服务：

```bash
python app.py
```

### 4.4 启动前端

回到项目根目录运行：

```bash
npm run dev
```

## 5. 界面说明

### 5.1 顶部工具栏

顶部区域提供以下能力：

- 中英文切换
- 导出 Notebook
- 生成假设与稳健性报告
- 导入数据
- 新建手动节点
- 撤销操作

### 5.2 画布操作

- 鼠标滚轮：滚动画布
- `Ctrl` / `Cmd` + 滚轮：缩放画布
- 按住空格或 `Alt` + 左键拖动：平移画布
- 拖动节点标题栏：移动节点
- 点击空白区域：取消选中

## 6. 典型使用流程

### 6.1 导入数据

点击 `Import` 导入 CSV 文件。导入成功后会生成一个 `Dataset` 节点，显示数据名称、样本量和变量列表。

### 6.2 分配变量角色

在数据集节点中，点击变量后方的角色徽章，可以在以下角色之间循环切换：

- `T`：Treatment
- `Y`：Outcome
- `C`：Confounder
- `M`：Mediator
- `I`：Instrument
- `K`：Collider
- `○`：未分配

这一步会直接影响后续因果图推断与效应估计。

### 6.3 自动生成探索分析

点击数据集节点中的 `Auto EDA`，系统会根据字段信息生成一组图表节点。常见包括：

- 散点图
- 分布图
- 基于数据关系的初步洞察

这些图表会以新的 `Visualization` 节点出现在数据集节点右侧，并自动建立连线。

### 6.4 生成因果图

点击数据集节点中的 `Suggest DAG`，系统会结合变量角色和上下文生成 `Causal Graph` 节点。该节点展示变量之间的因果关系，并附带简要说明与建议动作。

### 6.5 估计因果效应

点击因果图节点中的 `Estimate Causal Effect (ATE)`，系统会生成一个 `Insight` 结果节点，通常包含：

- 调整集
- ATE 点估计
- 置信区间
- p 值
- 公式
- 异质性建议

### 6.6 添加人工研究笔记

点击 `Manual Node` 可以新建手动节点。双击节点内容即可编辑。

当笔记内容较完整时，系统会自动分析该笔记，并尝试生成：

- 建议动作
- 后续分析入口
- 可继续展开的新结论节点

## 7. 节点交互说明

### 7.1 节点如何与其他节点交互

EchoForge 中节点之间并不是直接相互调用，而是通过统一的事件机制与画布状态协同工作：

1. 节点内部触发操作，例如编辑、点击建议、点击图表点位
2. 节点把操作上抛为 `onAction(action, payload)`
3. 上层容器处理动作并更新 `nodes` / `edges`
4. 画布根据最新状态重新渲染节点和连线

### 7.2 手动节点如何参与分析

手动节点可以通过以下方式和其他节点产生联系：

- 作为研究笔记被收集到 AI 分析上下文中
- 触发建议动作并生成新的分析节点
- 在笔记中通过 `@节点名称` 引用其他节点
- 点击引用后自动聚焦到被提及节点

### 7.3 图表节点交互

图表节点支持点击数据点。点击后系统会：

1. 弹出异常点分析浮层
2. 调用 AI 解释该点的可能意义
3. 可将解释固定为新的 `Insight` 节点

### 7.4 分叉分析

任意支持 `Fork` 的节点都可以创建分析分支。分叉后系统会复制当前节点及其下游依赖，并创建一条 `fork` 连线，方便保留不同假设下的分析路径。

## 8. 输出能力

### 8.1 Notebook 导出

点击顶部 `Notebook` 可生成可复现的分析源码，用于沉淀研究过程。

### 8.2 报告导出

点击顶部 `Report` 可生成假设与稳健性报告，用于总结：

- 关键研究假设
- 因果识别思路
- 风险点与后续建议

### 8.3 单节点代码查看

节点底部的 `Code` 按钮会为当前节点生成对应代码，便于复现实验或迁移到外部分析环境。

## 9. 使用建议

- 先导入数据，再分配关键变量角色，最后生成 DAG
- 手动节点尽量写成完整句子，便于 AI 理解上下文
- 重要结论建议通过 `Fork` 保留不同分支，避免覆盖原始路径
- 图表节点中的异常点诊断适合用于发现数据质量问题和潜在机制
- 估计结果应结合调整集与假设共同解读，不建议只看 p 值

## 10. 常见问题

### 10.1 点击按钮没有返回结果

先检查以下几项：

- Ollama 是否已启动
- 后端 `server/app.py` 是否已运行
- `.env` 中模型名称是否正确
- 前端请求地址是否能连到 `/api`

### 10.2 导入数据后图表为空

可能原因：

- CSV 首行表头格式不规范
- 某些字段不是数值，无法用于散点或分布图
- 数据列名与自动推断结果不匹配

### 10.3 手动节点没有触发建议动作

当前逻辑下，手动笔记通常需要有一定长度和明确含义，系统才会自动分析并给出建议。

### 10.4 AI 助手为什么显示开发中

`Ask AI` 入口当前仍是占位能力，仓库中还没有完整对话式节点问答流程。

## 11. 项目结构

主要目录如下：

```text
src/echoforge/      前端核心画布与节点系统
server/             Python 后端与模型服务
test_data.csv       示例数据
```

其中：

- `src/echoforge/EchoForge.tsx`：主应用与节点动作调度
- `src/echoforge/NodeViews.tsx`：各类节点视图
- `src/echoforge/InfiniteCanvas.tsx`：无限画布与连线渲染
- `src/echoforge/Ollama.ts`：前端到后端的 AI 请求封装
